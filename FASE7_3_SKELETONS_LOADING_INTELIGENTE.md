Objetivo: melhorar a percepção de performance sem alterar regras de negócio, Supabase, UX principal ou responsividade validada.

Aplicado:
- criação de `src/components/feedback/Skeletons.jsx`;
- skeletons para resumo/dashboard;
- skeletons para lista de contas;
- skeletons para painel/lista de notas;
- substituição do texto simples `Carregando...` por placeholders visuais;

Não alterado:
- regras financeiras;
- salvar/editar contas;
- recorrência;
- notas/lixeira;
- auth;
- Supabase;
- layout base validado.

Validação recomendada:
